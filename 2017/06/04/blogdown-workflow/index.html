Blogdown workflow

2017/06/04

I’ve relaunched my blog after a long hiatus. The reboot gave me a chance to learn a new tool–blogdown–and it forced me to revisit some old posts from my former blog platform. I wrote a lot of poetry in the last couple of years, most of it sparked by various personal events. Not all of it sounds as good to me now as it did then, but I avoided the temptation to edit my past. Somehow, I suspect that the future me will be glad for the reminders about what these events meant to me then, especially after they’ve faded.

The current workflow is a bit cumbersome, but it works for now. It’s something to tweak going forward. Here are the steps:

  1. Write new posts in blogdown using the blogdown::new_post() command.
  2. Build the site using the blogdown::build_site() command.
  3. Preview the site using blogdown::serve_site().
  4. Commit the new site once it passes muster.
  5. Copy the contents of the public/ directory to the rogilmore.github.io repo. This repo is set up as the CNAME for www.rick-gilmore.com using the GitHub pages facility. So, a few moments after I commit and push the files, they’re live.

I have a lot to learn about the hugo blog engine, and I suspect I will tweak the blog’s theme. But, as I mentioned earlier, I like the fact that the blog uses the RStudio family of tools which have become like an extra appendage. One of the charms of reviewing some of my old posts was seeing how that itself has evolved.
